File Control e I/O Avanzato in C

[Contesto: Gestione FIFO, pipe, read/write, poll(), fcntl(), dup2(), file sparsi, locking cooperativo e I/O asincrono]

1. APERTURA FILE (open)

Restituisce un File Descriptor (FD) intero. Se fallisce, ritorna -1.

  • Bloccante (Default):
    int fd = open("/tmp/fifo", O_RDONLY);
  • Non Bloccante:
    int fd = open("/tmp/fifo", O_RDONLY | O_NONBLOCK);

2. READ & WRITE SU FIFO

Sintassi base: ssize_t n = read(fd, &buf, sizeof(buf));

Scenario Bloccante Non Bloccante (O_NONBLOCK)
Read: Dati presenti Ritorna byte letti Ritorna byte letti
Read: Niente dati Si blocca in attesa Ritorna -1, errno=EAGAIN
Read: Nessun writer Ritorna 0 (EOF) Ritorna -1, poll genera POLLHUP
Write: Buffer pieno Si blocca in attesa Ritorna -1, errno=EAGAIN

3. MULTIPLEXING CON POLL()

Evita il busy waiting su file non bloccanti.

#include <poll.h>
struct pollfd poll_set[NUM_FILE];

for (int i = 0; i < NUM_FILE; i++) {
    poll_set[i].fd = fds[i];     // FD da monitorare
    poll_set[i].events = POLLIN; // Eventi da ascoltare
    poll_set[i].revents = 0;     // Output del kernel
}

Usare SEMPRE l'AND bit a bit (&) per controllare revents, MAI l'uguaglianza (==).
La poll() va inserita in un loop principale.

int ready = poll(poll_set, NUM_FILE, -1); // -1 = attesa infinita

for (int i = 0; i < NUM_FILE; i++) {
    if (poll_set[i].fd == -1) continue;

    // 1. Disconnessione
    if (poll_set[i].revents & POLLHUP) {
        close(poll_set[i].fd);
        poll_set[i].fd = -1;
        continue;
    }

    // 2. Dati Pronti (Loop per svuotare il buffer non bloccante)
    if (poll_set[i].revents & POLLIN) {
        ssize_t nread;
        int value;
        while ((nread = read(poll_set[i].fd, &value, sizeof(value))) > 0) {
            // Usa i dati
        }
        if (nread == -1 && errno != EAGAIN) {
            // Gestione vero errore
        }
    }
}

4. MANIPOLAZIONE DEI FLAG E I/O NON BLOCCANTE (fcntl)

Se un file descriptor è già stato aperto (o ereditato, come nei casi standard o nelle pipe), non è possibile cambiare la modalità di blocco direttamente tramite open(). Si usa la system call fcntl() con i comandi F_GETFL (prendi i flag correnti) e F_SETFL (imposta i nuovi flag).

#include <fcntl.h>
#include <errno.h>

// 1. Recupera i flag correnti associati al file descriptor
int flags = fcntl(fd, F_GETFL);
if (flags == -1) { perror("fcntl F_GETFL"); exit(1); }

// 2. Modifica i flag aggiungendo O_NONBLOCK tramite OR bit a bit (|)
if (fcntl(fd, F_SETFL, flags | O_NONBLOCK) == -1) {
    perror("fcntl F_SETFL");
    exit(1);
}

Quando un file è impostato come O_NONBLOCK:

  • read(): Se il canale è vuoto, non si blocca ma restituisce -1 impostando errno = EAGAIN (o EWOULDBLOCK).
  • write(): Se il buffer della pipe/socket è pieno, non si blocca ma restituisce -1 impostando errno = EAGAIN.
ssize_t n = read(fd, buf, sizeof(buf));
if (n < 0) {
    if (errno == EAGAIN || errno == EWOULDBLOCK) {
        // Il buffer è temporaneamente vuoto, non è un errore fatale
        printf("Nessun dato disponibile al momento, riprova più tardi.\n");
    } else {
        perror("Errore critico in lettura"); exit(1);
    }
}

5. REDIREZIONE DEI FILE DESCRIPTOR (dup2)

La system call dup2(int oldfd, int newfd) clona il file descriptor oldfd sul file descriptor newfd. Se newfd era già aperto, il kernel lo chiude automaticamente prima di effettuare la copia.

Questo meccanismo è la base di come la Shell implementa l'operatore >.

#include <unistd.h>
#include <fcntl.h>

// Apre un file di log. O_TRUNC svuota il file se esiste già, O_CREAT lo crea se manca
int logfd = open("output.log", O_WRONLY | O_CREAT | O_TRUNC, 0644);
if (logfd == -1) { perror("open"); return 1; }

/* Forza il file descriptor 1 (stdout) a puntare allo stesso file di logfd */
// Il kernel chiude automaticamente il vecchio stdout prima della sovrascrittura
if (dup2(logfd, 1) == -1) {
    perror("dup2"); return 1;
}
close(logfd); /* Il descrittore originale non serve più, usiamo l'1 clonato */

// Da questo momento in poi, qualsiasi printf scriverà nel file!
printf("Questo testo non andrà a schermo, ma dentro output.log\n");

6. FILE SPARSI (Sparse Files)

Un file sparso è un file che contiene dei "buchi" logici di dati (zeri completi) che non occupano spazio fisico reale sul disco fisso finché non vengono effettivamente sovrascritti. Si creano spostando il cursore molto in avanti con lseek() oltre la fine effettiva dei byte scritti.

#include <unistd.h>
#include <fcntl.h>

int fd = open("file_sparso.bin", O_CREAT | O_WRONLY, 0644);

// Sposta il cursore a 1 GB di distanza dall'inizio, lasciando un "buco" vuoto
off_t offset = lseek(fd, 1024LL * 1024 * 1024, SEEK_SET);
if (offset == (off_t)-1) { perror("lseek"); return 1; }

// Scrivendo anche un solo carattere alla fine, il file risulterà logicamente grande 1GB
write(fd, "A", 1); 
close(fd);
  • ls -lh file_sparso.bin → Mostra la dimensione logica (es. 1 GB).
  • du -sh file_sparso.bin → Mostra lo spazio reale effettivamente allocato sul disco (poche frazioni di KB).

7. ADVISORY FILE LOCKING (struct flock & fcntl)

Il locking tramite fcntl() è di tipo advisory (cooperativo). Significa che il Kernel non impedisce fisicamente a un processo malintenzionato di scrivere sul file; spetta ai programmi concorrenti cooperare richiedendo esplicitamente il lock prima di accedere alla risorsa.

struct flock lock;
lock.l_type   = F_WRLCK;   // Tipo: F_RDLCK (Condiviso/Lettura), F_WRLCK (Esclusivo/Scrittura), F_UNLCK (Sblocca)
lock.l_whence = SEEK_SET;  // Punto di riferimento dell'offset (SEEK_SET, SEEK_CUR, SEEK_END)
lock.l_start  = 0;         // Offset iniziale in byte relativo a l_whence
lock.l_len    = 0;         // Lunghezza della sezione da bloccare. 0 = fino alla fine del file (EOF)
lock.l_pid    = 0;         // Compilato dal Kernel con F_GETLK per indicare chi possiede il lock
  • F_SETLK: Tenta di acquisire il lock. Se il file è già bloccato da un altro processo, fallisce immediatamente restituendo -1 e impostando errno = EACCES o EAGAIN.
  • F_SETLKW: (Wait) Se incontra un lock in conflitto, mette a dormire il processo finché la risorsa non torna libera.
#include <fcntl.h>
#include <unistd.h>

// Per poter inserire un Write Lock (F_WRLCK), il file DEVE essere aperto almeno in O_WRONLY o O_RDWR
int fd = open("data.txt", O_RDWR);
struct flock lock = {0};
lock.l_type = F_WRLCK;    // Richiesta di Lock Esclusivo (Write Lock)
lock.l_whence = SEEK_SET; // Partiamo dall'inizio del file

// 1. Tentativo NON bloccante
if (fcntl(fd, F_SETLK, &lock) == -1) {
    perror("Impossibile bloccare, file già occupato");
    
    // Curiosità d'esame: scopriamo chi sta bloccando il file
    fcntl(fd, F_GETLK, &lock); // Sovrascrive la struttura lock con le informazioni del lock attivo
    printf("Il file è attualmente bloccato dal processo con PID: %d\n", lock.l_pid);
    
    close(fd);
    return 1;
}

/* --- INIZIO SEZIONE CRITICA (Lavoro sul file in sicurezza) --- */
printf("Lock acquisito con successo dal processo %d\n", getpid());
sleep(5); 
/* --- FINE SEZIONE CRITICA --- */

// 2. Rilascio del Lock
lock.l_type = F_UNLCK; // Imposta il tipo a sblocco
fcntl(fd, F_SETLK, &lock);
close(fd);

8. I/O ASINCRONO GUIDATO DAI SEGNALI (O_ASYNC)

L'I/O asincrono permette a un processo di continuare a svolgere i propri calcoli in user-space. Il Kernel invierà un segnale SIGIO al processo non appena sul file descriptor diventano disponibili nuovi dati da leggere.

Richiede 3 passaggi fondamentali:

  1. Assegnare un gestore personalizzato per il segnale SIGIO.
  2. Impostare la proprietà del file descriptor (F_SETOWN) indicando quale PID deve ricevere il segnale.
  3. Attivare il flag O_ASYNC (e solitamente anche O_NONBLOCK) sul file.
#include <stdio.h>
#include <fcntl.h>
#include <signal.h>
#include <unistd.h>

// Flag asincrono globale: modificato dall'handler, letto dal main
volatile sig_atomic_t got_sigio = 0;
void sigio_handler(int signo) { got_sigio = 1; }

int main() {
    int fd = open("/tmp/myfifo", O_RDONLY | O_NONBLOCK);

    // Passaggio 1: Installa il Signal Handler per SIGIO
    struct sigaction sa = {0};
    sa.sa_handler = sigio_handler;
    sigaction(SIGIO, &sa, NULL);

    // Passaggio 2: Associa il file descriptor al PID corrente (F_SETOWN)
    // Indica al kernel a chi mandare il segnale SIGIO quando il file è pronto
    fcntl(fd, F_SETOWN, getpid());

    // Passaggio 3: Attiva lo smistamento asincrono aggiungendo O_ASYNC
    int flags = fcntl(fd, F_GETFL);
    fcntl(fd, F_SETFL, flags | O_ASYNC);

    printf("Pronto a fare altro in attesa dei dati...\n");
    
    while(1) {
        if (got_sigio) {
            got_sigio = 0; // Resetta il flag
            char buf[256];
            ssize_t n = read(fd, buf, sizeof(buf));
            if (n > 0) printf("Dati asincroni ricevuti: %.*s\n", (int)n, buf);
        }
        // Il programma può fare altri calcoli qui in totale libertà!
        usleep(500000); 
    }
    return 0;
}
