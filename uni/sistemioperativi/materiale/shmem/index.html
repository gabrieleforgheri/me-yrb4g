Shared Memory, mmap() e shm_open().

Memoria Condivisa (Shared Memory)

[Context: Condivisione di un'area di memoria (RAM) ad altissime prestazioni tra processi. Richiede sincronizzazione manuale (tramite flag, semafori o segnali) per evitare race conditions.]

0. ⚠️ ATTENZIONE: COMPILAZIONE SUI PC DEL LABORATORIO

Sulle macchine del laboratorio virtuale (e su molte distribuzioni Linux), per usare la memoria condivisa POSIX con nome (es. shm_open, shm_unlink) DEVI compilare linkando la libreria runtime del C (librt).

Comando obbligatorio all'esame:

gcc -o eseguibile sorgente.c -lrt

Se dimentichi -lrt, il linker ti darà un errore del tipo "undefined reference to 'shm_open'" e la compilazione fallirà.

1. SHARED MEMORY ANONIMA (mmap senza file)

Si usa ESCLUSIVAMENTE tra processi legati da parentela (es. padre e figlio tramite fork()). La memoria viene mappata direttamente senza appoggiarsi a un file nel filesystem.

A. Creazione (Prima della fork())

#include <sys/mman.h>

// Alloca lo spazio per un singolo intero (o per una struct)
int *shared = mmap(NULL,            // L'indirizzo lo decide il kernel
                   sizeof(int),     // Dimensione dell'area in byte
                   PROT_READ | PROT_WRITE, // Permessi di Lettura e Scrittura
                   MAP_SHARED | MAP_ANONYMOUS, // Condivisa e senza file (Anonima)
                   -1,              // File Descriptor ignorato (-1) in anonima
                   0);              // Offset ignorato (0) in anonima

if (shared == MAP_FAILED) { perror("mmap"); exit(1); } // Controllo errori

*shared = 0; // Inizializzazione prima della fork

B. Rimozione (munmap)

Dopo aver usato la memoria, va liberata usando munmap(). Viene comunque liberata automaticamente alla terminazione del processo.

munmap(shared, sizeof(int));

2. SHARED MEMORY CON NOME (POSIX Shared Memory)

Si usa per far comunicare processi scorrelati (es. due terminali diversi lanciati separatamente). Crea un "file speciale" temporaneo visibile in /dev/shm.

A. Processo PRODUTTORE (Chi la crea)

#include <sys/mman.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <sys/shm.h> // Richiede -lrt in compilazione

const int SIZE = 4096;
const char *name = "/MIA_SHM"; // Il nome convenzionale inizia spesso con '/'

// 1. Apre/Crea l'oggetto di memoria condivisa
int fd = shm_open(name, O_CREAT | O_RDWR, 0666);
if (fd == -1) { perror("shm_open"); exit(1); }

// 2. Imposta la dimensione (Obbligatorio alla creazione!)
ftruncate(fd, SIZE);

// 3. Mappa l'oggetto in memoria
char *ptr = (char *)mmap(0, SIZE, PROT_WRITE, MAP_SHARED, fd, 0);

// 4. Ci scrive dentro usando normali funzioni del C
sprintf(ptr, "Hello World!");

B. Processo CONSUMATORE (Chi la legge)

// 1. Apre l'oggetto ESISTENTE in sola lettura
int fd = shm_open(name, O_RDONLY, 0666);

// 2. Mappa l'oggetto in memoria (Non serve ftruncate qui!)
char *ptr = (char *)mmap(NULL, SIZE, PROT_READ, MAP_SHARED, fd, 0);

// 3. Legge e usa i dati
printf("Letto: %s\n", ptr);

// 4. Distruzione finale (Solo un processo deve farlo!)
shm_unlink(name); // Rimuove l'oggetto da /dev/shm

Nota sulla persistenza: Se non chiami shm_unlink, l'oggetto sopravvive alla terminazione dei processi e occupa RAM finché non riavvii il PC.

3. SINCRONIZZAZIONE: L'USO DELLE struct

La memoria condivisa è "troppo veloce". Se due processi ci scrivono assieme, si corrompe (Race Condition). Per ovviare, si mappa un'intera struct contenente campi di stato.

// Esempio di struttura condivisa
typedef struct {
    int status; // Flag (es. 0 = VUOTO, 1 = DATI_PRONTI, 2 = RISULTATO_PRONTO) 
    char input[1024];
    char output[1024];
} shared_data;

// Mappatura
shared_data *shm = mmap(NULL, sizeof(shared_data), PROT_READ | PROT_WRITE, MAP_SHARED | MAP_ANONYMOUS, -1, 0);

Per attendere che i dati siano pronti, un processo può:

  • Fare "Busy Waiting" ciclando passivamente con una sleep(1) finché lo stato non cambia.
  • Usare una pause() accoppiata a segnali (SIGUSR1, SIGUSR2) per risvegliarsi solo quando l'altro processo lo avvisa.
