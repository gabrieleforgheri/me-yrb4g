Shell Scripting (Bash)

[Context: Automazione di sistema, gestione di file e processi, redirezione dell'I/O e filtri di testo in ambiente Unix/Linux.]

1. BASI E SINTASSI FONDAMENTALE

A. L'Intestazione (Shebang)

Ogni script Bash deve iniziare con lo "shebang" per indicare al sistema quale interprete usare.

#!/bin/bash

Ricorda di dare i permessi di esecuzione allo script: chmod +x script.sh

B. Variabili e Assegnazione

In Bash, non ci devono essere spazi attorno al segno di uguale (=).

NOME="Mario"      # CORRETTO
NOME = "Mario"    # ERRORE! Bash pensa che NOME sia un comando
let NUM=5+3       # Operazioni matematiche base (vecchio stile)
NUM=$((5 + 3))    # Operazioni matematiche (stile moderno raccomandato)

C. Quoting (Apici vs Virgolette)

  • ' ' (Singoli): Testo letterale. Nessuna espansione viene fatta.
  • " " (Doppi): Testo espanso. Risolve variabili e command substitution.
  • ` ` (Backtick) o $(): Command Substitution. Esegue il comando e ne restituisce l'output.
echo "Ciao $NOME"      # Stampa: Ciao Mario
echo 'Ciao $NOME'      # Stampa: Ciao $NOME
OGGI=$(date +%F)       # Salva l'output del comando 'date' nella variabile

2. PARAMETRI SPECIALI E VARIABILI DI STATO

Fondamentali per gestire gli argomenti passati allo script da riga di comando.

Variabile Significato
$0Nome dello script stesso.
$1, $2...Primo, secondo, ecc... argomento passato allo script.
$#Numero totale di argomenti passati.
$@Tutti gli argomenti passati come array (lista separata da spazi).

3. REDIREZIONE E PIPE (I/O)

Unix gestisce l'I/O tramite tre stream: stdin (0), stdout (1), stderr (2).

OperatoreAzioneEsempio
>Redirige lo stdout su file (sovrascrive)ls > lista.txt
>>Redirige lo stdout su file (appende in coda)echo "ciao" >> log.txt
<Legge da un file verso lo stdinsort < lista.txt
2>Redirige solo gli errori (stderr)find / -name "test" 2> errori.log
&>Redirige sia stdout che stderrcomando &> out_completo.log

4. STRUTTURE CONDIZIONALI (if, test, [ ])

In Bash, lo spazio tra le parentesi quadre [ ] e la condizione è OBBLIGATORIO.

if [ condizione ]; then
    # comandi
elif [ condizione2 ]; then
    # comandi
else
    # comandi
fi

Operatori di Test Più Comuni

  • Sui Numeri (Attenzione: si usano le lettere, non > / <)
    • -eq / -ne: Uguale a / Diverso da
    • -gt / -ge: Maggiore di / Maggiore o uguale
    • -lt / -le: Minore di / Minore o uguale
  • Sulle Stringhe
    • = o ==: Stringhe identiche
    • !=: Stringhe diverse
    • -z "$STR": Vero se la stringa è vuota (lunghezza zero)
    • -n "$STR": Vero se la stringa NON è vuota
  • Sui File e Directory
    • -f file: Vero se è un file regolare.
    • -d dir: Vero se è una directory.
    • -e path: Vero se il percorso esiste.
    • -r file: Vero se il file è leggibile.
    • -x file: Vero se il file è eseguibile.

Esempio combinato:

if [ -f "$1" ] && [ -r "$1" ]; then
    echo "Il file passato come primo argomento esiste ed è leggibile."
fi

5. CICLI (for, while, until)

Ciclo for

# Iterare su una lista di stringhe
for FRUTTO in Mela Pera Banana; do
    echo "Mi piace la $FRUTTO"
done

# Iterare su file di una directory
for FILE in *.txt; do
    echo "Elaboro il file: $FILE"
done

# Ciclo for stile C
for (( i=0; i<10; i++ )); do
    echo "Contatore: $i"
done

Ciclo while e until

# WHILE: cicla finché la condizione è VERA
CONTATORE=0
while [ $CONTATORE -lt 5 ]; do
    echo "Contatore: $CONTATORE"
    CONTATORE=$((CONTATORE + 1))
done

# UNTIL: cicla finché la condizione è FALSA
until [ -f "flag.txt" ]; do
    echo "Aspetto che qualcuno crei flag.txt..."
    sleep 2
done

Leggere un file riga per riga

while IFS= read -r RIGA; do
    echo "Riga letta: $RIGA"
done < "miofile.txt"

6. FILTRI E MANIPOLAZIONE TESTO (The Core Unix Tools)

Questi strumenti, combinati con le | (pipe), sono il cuore di Bash.

  • grep (Ricerca di pattern)
    • grep "errore" file.log: Trova "errore".
    • grep -i "errore": Case-insensitive.
    • grep -v "info": Inverte il match (mostra le righe che NON contengono "info").
    • grep -c "errore": Conta il numero di righe.
  • awk (Elaborazione basata su colonne/campi)
    • awk '{print $1, $3}' file.txt: Stampa prima e terza colonna.
    • awk -F ":" '{print $1}' /etc/passwd: Usa i due punti come separatore.
    • awk '{sum += $1} END {print sum}': Somma tutti i valori.
  • sed (Stream Editor - Sostituzione)
    • sed 's/vecchio/nuovo/' file.txt: Sostituisce la prima occorrenza.
    • sed 's/vecchio/nuovo/g' file.txt: Sostituzione globale.
    • sed '/PATTERN/d' file.txt: Elimina le righe che contengono il pattern.
  • Altri filtri utili
    • cut -d',' -f2 file.csv: Estrae la colonna 2 da un CSV usando la virgola.
    • sort -n: Ordina in modo numerico.
    • uniq / uniq -c: Rimuove righe duplicate / le conta.

7. PROCESSI IN BACKGROUND E SEGNALI (trap)

./script_lungo.sh &
PID_LUNGO=$!      # Salva il PID dell'ultimo processo in background
wait $PID_LUNGO   # Il padre si sospende finché questo figlio non termina

La gestione dei Segnali (trap)

# Definiamo la funzione di cleanup
cleanup() {
    echo "Rilevato SIGINT (Ctrl+C). Pulizia dei file temporanei..."
    rm -f /tmp/temp_*.log
    exit 1
}

# Installiamo il trap per intercettare SIGINT (2) e SIGTERM (15)
trap cleanup SIGINT SIGTERM

echo "Script in esecuzione. Premi Ctrl+C per testare..."
while true; do
    sleep 1
done

8. FUNZIONI

Usa la keyword local dentro le funzioni per isolare le variabili!

mia_funzione() {
    local PARAM_1=$1  
    local PARAM_2=$2  
    
    echo "Ho ricevuto: $PARAM_1 e $PARAM_2"
    
    # In Bash si usa 'echo' per restituire valori catturabili dal chiamante
    echo "RISULTATO"
}

# Chiamata di funzione e cattura del risultato
RET_VAL=$(mia_funzione "Ciao" "Mondo")
echo "La funzione ha restituito: $RET_VAL"

Esempio 4: Array Associativi e Xargs

#!/bin/bash
# Dichiara un array associativo (chiave-valore)
declare -A EXT_COUNTS

for file in *; do
    [[ -f "$file" ]] || continue
    ext=$(echo $file | cut -d. -f2-)
    if [[ -z $ext ]]; then ext="no_extension"; fi
    
    ((EXT_COUNTS[$ext]++))
done

# Iterazione sulle chiavi
for ext in "${!EXT_COUNTS[@]}"; do
    mkdir -p "$ext"
    # L'uso di xargs -0 con printf '\0' permette di gestire file con spazi
    for file in *.$ext; do
        printf '%s\0' "$file"
    done | xargs -0 -I{} mv {} $ext
done

#!/bin/bash
audit_file() {
    file="$1"
    perms=$(stat -c "%a" "$file")
    world_perms="${perms: -1}"

    if [ "$world_perms" -ge 4 ]; then
        echo "[WARNING] $file is world-readable!"
        chmod o-r "$file" 
    fi
}
export -f audit_file

sensitive_files=("/etc/passwd" "/etc/shadow")
printf "%s\0" "${sensitive_files[@]}" | xargs -0 -I {} bash -c 'audit_file "{}"'

Esempio 6: Gestione Log Temporale (touch -d)

#!/bin/bash
AGE_THRESHOLD="7 days ago"
SIZE_THRESHOLD=$((10 * 1024 * 1024)) # 10 MB

REF_FILE=".age_threshold_marker"
touch -d "$AGE_THRESHOLD" "$REF_FILE" 2>/dev/null

for log_file in *.log; do
    [ -f "$log_file" ] || continue
    # -ot confronta la data di modifica
    if [ "$log_file" -ot "$REF_FILE" ]; then
        file_size=$(stat -c "%s" "$log_file")
        if [ "$file_size" -gt "$SIZE_THRESHOLD" ]; then
            rm "$log_file"
        else
            gzip "$log_file"
        fi
    fi
done
rm -f "$REF_FILE"

Esempio 7: Controllo Processi (pgrep, renice)

#!/bin/bash
PROCESS_NAME="$1"
pids=$(pgrep -i "$PROCESS_NAME")

for pid in $pids; do
    if ! ps -p "$pid" > /dev/null 2>&1; then continue; fi
    
    read -p "Select priority for PID $pid [1) High, 2) Low]: " choice
    case "$choice" in
        1|"[Hh]igh") new_nice="-10" ;; 
        2|"[Ll]ow")  new_nice="10"  ;; 
        *) continue ;;
    esac
    
    renice -n "$new_nice" -p "$pid" > /dev/null 2>&1
done

Esempio 9: Monitoraggio SysFS

#!/bin/bash
CUR_FREQ_FILE="/sys/devices/system/cpu/cpu0/cpufreq/scaling_cur_freq"
OLD_FREQ=$(cat "$CUR_FREQ_FILE")

while true; do
    CUR_FREQ=$(cat "$CUR_FREQ_FILE")
    STABLE_THRESHOLD=$(( OLD_FREQ / 10 ))
    
    if (( CUR_FREQ <= OLD_FREQ + STABLE_THRESHOLD && CUR_FREQ >= OLD_FREQ - STABLE_THRESHOLD )) ; then
        TREND="Stable"
    elif (( CUR_FREQ > OLD_FREQ )); then
        TREND="Increasing"
    else
        TREND="Decreasing"
    fi

    printf "Frequenza: %s kHz | Trend: %s\n" "$CUR_FREQ" "$TREND"
    OLD_FREQ=$CUR_FREQ
    sleep 1
done

Esempio 11: Parsing Configurazione (sed)

#!/bin/bash
process_file() {
    local file_path="$1"
    
    sed -e 's/#.*//' -e 's/^[[:space:]]*$//' -e '/^$/d' "$file_path" | while read -r line; do
        include_target=$(echo "$line" | sed -n "s/^include[[:space:]]\+\(.\+\)/\1/p")
        
        if [[ -n $include_target ]]; then
            process_file "$include_target" 
        else
            echo "$line"
        fi
    done
}
process_file "$1"
