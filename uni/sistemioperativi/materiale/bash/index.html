1. BASI E SINTASSI FONDAMENTALE

A. L'Intestazione (Shebang)

Ogni script Bash deve iniziare con lo "shebang" per indicare al sistema quale interprete usare.

#!/bin/bash

Ricorda di dare i permessi di esecuzione allo script: chmod +x script.sh

B. Variabili e Assegnazione

In Bash, non ci devono essere spazi attorno al segno di uguale (=).

NOME="Mario"      # CORRETTO
NOME = "Mario"    # ERRORE! Bash pensa che NOME sia un comando
let NUM=5+3       # Operazioni matematiche base (vecchio stile)
NUM=$((5 + 3))    # Operazioni matematiche (stile moderno raccomandato)

C. Quoting (Apici vs Virgolette)

  • ' ' (Singoli): Testo letterale. Nessuna espansione viene fatta.
  • " " (Doppi): Testo espanso. Risolve variabili e command substitution.
  • ` ` (Backtick) o $(): Command Substitution. Esegue il comando e ne restituisce l'output.
echo "Ciao $NOME"      # Stampa: Ciao Mario
echo 'Ciao $NOME'      # Stampa: Ciao $NOME
OGGI=$(date +%F)       # Salva l'output del comando 'date' nella variabile

4. STRUTTURE CONDIZIONALI (if, test, [ ])

In Bash, lo spazio tra le parentesi quadre [ ] e la condizione è OBBLIGATORIO.

if [ condizione ]; then
    # comandi
elif [ condizione2 ]; then
    # comandi
else
    # comandi
fi

Operatori di Test Più Comuni

  • Sui Numeri (Attenzione: si usano le lettere, non > / <)
    • -eq / -ne: Uguale a / Diverso da
    • -gt / -ge: Maggiore di / Maggiore o uguale
    • -lt / -le: Minore di / Minore o uguale
  • Sulle Stringhe
    • = o ==: Stringhe identiche
    • !=: Stringhe diverse
    • -z "$STR": Vero se la stringa è vuota (lunghezza zero)
    • -n "$STR": Vero se la stringa NON è vuota
  • Sui File e Directory
    • -f file: Vero se è un file regolare.
    • -d dir: Vero se è una directory.
    • -e path: Vero se il percorso esiste.
    • -r file: Vero se il file è leggibile.
    • -x file: Vero se il file è eseguibile.

Esempio combinato:

if [ -f "$1" ] && [ -r "$1" ]; then
    echo "Il file passato come primo argomento esiste ed è leggibile."
fi

5. CICLI (for, while, until)

Ciclo for

# Iterare su una lista di stringhe
for FRUTTO in Mela Pera Banana; do
    echo "Mi piace la $FRUTTO"
done

# Iterare su file di una directory
for FILE in *.txt; do
    echo "Elaboro il file: $FILE"
done

# Ciclo for stile C
for (( i=0; i<10; i++ )); do
    echo "Contatore: $i"
done

Ciclo while e until

# WHILE: cicla finché la condizione è VERA
CONTATORE=0
while [ $CONTATORE -lt 5 ]; do
    echo "Contatore: $CONTATORE"
    CONTATORE=$((CONTATORE + 1))
done

# UNTIL: cicla finché la condizione è FALSA
until [ -f "flag.txt" ]; do
    echo "Aspetto che qualcuno crei flag.txt..."
    sleep 2
done

Leggere un file riga per riga

while IFS= read -r RIGA; do
    echo "Riga letta: $RIGA"
done < "miofile.txt"

6. FUNZIONI E SCOPE DELLE VARIABILI

Definizione e Passaggio dei Parametri

Le funzioni in Bash sono blocchi di comandi riutilizzabili isolati in un nome logico. Importante: i parametri passati ad una funzione non coincidono con i parametri dello script principale. All'interno della funzione, i registri $1, $2, $# si azzerano e fanno riferimento esclusivamente agli argomenti della funzione stessa.

Uso Obbligatorio di local

Di default, tutte le variabili in Bash sono globali. Se modifichi una variabile in una funzione, cambierà anche all'esterno, provocando bug catastrofici. È obbligatorio usare la parola chiave local per isolare lo scope all'interno del record di attivazione della funzione.

calcola_area() {
    local BASE=$1     # Scope locale, isolato dal resto dello script
    local ALTEZZA=$2
    local AREA=$((BASE * ALTEZZA))
    echo "$AREA"      # Le funzioni restituiscono dati scrivendo sullo stdout
}

# Invocazione della funzione (senza parentesi tonde!)
RISULTATO=$(calcola_area 10 5)

7. STRUTTURE DATI AVANZATE: ARRAY INDIZZATI E ASSOCIATIVI

Array Indizzati (Liste Standard)

NOMI=("Alice" "Bob" "Charlie") # Creazione dell'array
echo "${NOMI[0]}"              # Accesso al primo elemento (Indice 0)
echo "${#NOMI[@]}"             # Restituisce la lunghezza dell'array

Array Associativi (Chiave-Valore / Mappe)

Gli array associativi permettono di mappare chiavi testuali a valori. Richiedono obbligatoriamente una dichiarazione esplicita tramite il comando builtin declare -A.

# 1. Dichiarazione esplicita obbligatoria
declare -A MAPPA_ESTENSIONI

# 2. Popolamento e assegnazione
MAPPA_ESTENSIONI[txt]=15
MAPPA_ESTENSIONI[pdf]=42

# 3. Iterazione sulle CHIAVI dell'array (Uso del punto esclamativo)
for ext in "${!MAPPA_ESTENSIONI[@]}"; do
    echo "Estensione: $ext -> File spostati: ${MAPPA_ESTENSIONI[$ext]}"
done

8. STRUMENTI DI ELABORAZIONE TESTO (SWISS ARMY KNIVES)

grep (Global Regular Expression Print)

Filtra flussi testuali estraendo esclusivamente le righe che corrispondono ad un pattern o espressione regolare.

  • -E: Attiva le Espressioni Regolari Estese (ERE) evitando l'escape di caratteri speciali come ( ) e |.
  • -i: Disattiva la sensibilità alle maiuscole/minuscole (case-insensitive).
  • -v: Inverte il filtro, mostrando solo le righe che non corrispondono al pattern.
  • -c: Restituisce il conteggio numerico puro delle linee corrispondenti.

sed (Stream Editor)

Modifica flussi di testo in transito riga per riga basandosi su comandi di editing.

  • Sostituzione di Base (s/pattern/replacement/): Cambia la prima occorrenza del pattern trovata in ogni riga.
  • Sostituzione Globale (s/pattern/replacement/g): Cambia tutte le occorrenze presenti in ogni riga.
  • Capture Groups ed ERE (sed -E): Permette di isolare parti del testo tramite parentesi tonde e riutilizzarle nella stringa di rimpiazzo mediante i puntatori sequenziali \1, \2, \3.
# Estrae l'IP (\1), la Data (\2) e l'Errore (\3) da un log HTTP riformattando la riga
sed -E 's/^([^ ]+) .* \[(.*)\] ".*" (404|500) .*/Error \3 from IP \1 at time \2/' log.txt
  • Cancellazione Righe (/pattern/d): Rimuove completamente dal flusso le righe che corrispondono al criterio impostato.

awk (Data Extraction & Processing Language)

awk interpreta le righe di testo come un database strutturato composto da record (righe) e campi (colonne). Di default, usa lo spazio bianco o il tabulatore come separatore di colonna.

  • Variabili di Colonna: $0 indica l'intera riga corrente; $1, $2 indicano rispettivamente la prima e la seconda colonna estratta.
  • Flag -F: Specifica un delimitatore personalizzato differente dallo spazio.
# Estrae l'username (colonna 1) dal file degli utenti di sistema usando ':' come separatore
awk -F ":" '{ print $1 }' /etc/passwd

Blocchi Temporali (BEGIN ed END):

  • BEGIN { ... }: Esegue il blocco di codice una volta sola prima di iniziare a leggere il file.
  • END { ... }: Esegue il codice al termine della lettura di tutte le righe (perfetto per stampare medie o somme accumulate).
# Somma i valori presenti nella prima colonna di un file e stampa il totale complessivo
awk '{ sum += $1 } END { print "Totale Accumulato:", sum }' dati.txt

L'opzione -E (usata con grep -E o sed -E) abilita le Extended Regular Expressions. Questo permette di usare operatori logici, quantificatori e gruppi senza doverli precedere dal carattere di escape \.

Esempi Pratici e Pattern Ricorrenti da Esame

1. Trovare specifiche estensioni di file (OR testuale):

# Trova tutte le righe che terminano con .png, .jpg o .gif
grep -E '\.(png|jpg|gif)$' lista_file.txt

(Nota: il punto . ha il backslash davanti \. perché altrimenti indicherebbe "qualsiasi carattere", mentre noi vogliamo cercare proprio il carattere punto testuale).

2. Ricerca di un Indirizzo IP (Quantificatori combinati):
Un IP è formato da 1 a 3 numeri, seguiti da un punto, il tutto ripetuto.

# Trova sequenze di numeri tipo 192.168.1.1
grep -E '^[0-9]{1,3}(\.[0-9]{1,3}){3}$' log.txt

3. Cattura e Inversione di un formato Data con sed -E:
Se hai un file con date in formato YYYY-MM-DD e le vuoi convertire in DD/MM/YYYY.
I gruppi racchiusi tra parentesi tonde () vengono memorizzati automaticamente da sed in variabili numerate \1, \2 e \3.

# Input: 2026-05-24
# Output desiderato: 24/05/2026

sed -E 's/^([0-9]{4})-([0-9]{2})-([0-9]{2})$/\3\/\2\/\1/' date.txt

Spiegazione del comando sed:
([0-9]{4}) -> Cattura 4 numeri (L'anno). Diventa \1
- -> Match letterale del trattino
([0-9]{2}) -> Cattura 2 numeri (Il mese). Diventa \2
\3\/\2\/\1 -> Scrive il terzo gruppo, poi una barra (escapata \/), il secondo, barra, il primo.

4. Estrapolare un Indirizzo IP con sed -E:
In questo esempio catturiamo i quattro gruppi numerici (ottetti) di un indirizzo IP, ignorando tutto il testo che lo precede e lo segue.

sed -E "s/^.* ([0-9]{1,3})\.([0-9]{1,3})\.([0-9]{1,3})\.([0-9]{1,3}) .*/\1.\2.\3.\4/" network.log

Spiegazione del comando:
^.* -> Ignora qualsiasi testo dall'inizio della riga fino allo spazio che precede l'IP.
([0-9]{1,3}) -> Cattura da 1 a 3 cifre. Siccome ci sono quattro gruppi parentetici, creano i puntatori \1, \2, \3, \4.
\. -> Fai il match letterale del punto (deve essere escapato).
La sostituzione \1.\2.\3.\4 rimpiazza l'intera riga lasciando solo l'IP estratto.

5. Validare un indirizzo MAC:
I MAC Address sono formati da coppie di caratteri esadecimali separati da due punti (es. 00:1A:2B:3C:4D:5E).

grep -E '^([0-9A-Fa-f]{2}:){5}[0-9A-Fa-f]{2}$' network.log

Spiegazione: "Trova esattamente due caratteri esadecimali seguiti dai due punti, ripeti questo schema 5 volte, e concludi con altri due caratteri esadecimali".

APPROFONDIMENTO: Elaborazione di File CSV in Bash

I file CSV (Comma-Separated Values) sono il formato standard per l'esportazione di dati tabulari. In Bash, l'elaborazione di CSV "puliti" (dove il separatore non compare mai all'interno dei dati testuali) è un'operazione estremamente comune.

1. Estrazione rapida di colonne con cut

Il tool cut è lo strumento più veloce se devi semplicemente isolare una o più colonne, senza fare calcoli o logica complessa.

  • -d',' (Delimiter): Specifica la virgola come separatore.
  • -f (Fields): Specifica i numeri delle colonne (1-indexed).
# Estrae solo la prima e la terza colonna (es. Nome ed Email)
cut -d',' -f1,3 utenti.csv

# Estrae dalla colonna 2 fino alla fine della riga
cut -d',' -f2- utenti.csv

2. Il Ciclo while read con IFS

Se hai bisogno di iterare riga per riga e usare i valori delle colonne come variabili per altre operazioni di sistema (es. creare cartelle o utenti), la combinazione di IFS (Internal Field Separator) e read è la scelta migliore.

Pattern: Saltare l'header ed estrarre le variabili
Questo è un trucco molto elegante in Bash per ignorare la prima riga (l'intestazione delle colonne) e scorrere il resto del file:

#!/bin/bash

# Il blocco tra parentesi graffe redirige l'intero file al suo interno
{
    # 1. La prima chiamata a 'read' consuma l'intestazione e la ignora
    read header 
    
    # 2. Il ciclo consuma il resto del file
    # IFS=',' divide la riga. Le variabili prendono i valori delle colonne.
    while IFS=',' read -r username dipartimento quota; do
        
        echo "Creazione utente $username per il reparto $dipartimento..."
        # Esempio: useradd -m -g "$dipartimento" "$username"
        
    done
} < "utenti.csv"

3. La Potenza di awk sui CSV (Calcoli e Filtri)

Quando devi fare somme, calcolare medie o filtrare righe basandoti sul valore numerico di una colonna, awk è insuperabile.

  • -F',': Dice ad awk di usare la virgola come separatore di campo.
  • NR (Number of Record): È la variabile interna di awk che indica il numero di riga corrente. NR > 1 è il modo standard di awk per saltare l'intestazione.

Esempio A: Filtro basato su una colonna

# Stampa il Nome ($1) e il Ruolo ($2) solo se lo Stipendio ($4) è maggiore di 2000.
# NR > 1 ignora la prima riga (l'header).
awk -F',' 'NR > 1 && $4 > 2000 { print $1, "fa il", $2 }' dipendenti.csv

Esempio B: Somma e Calcolo Media

# Somma tutti i valori della colonna 3 (es. Ore lavorate) e alla fine ne calcola la media
awk -F',' '
    NR > 1 { 
        totale += $3 
        conteggio++ 
    } 
    END { 
        print "Ore totali:", totale
        print "Media ore per dipendente:", totale / conteggio
    }
' ore_lavoro.csv

4. Gestire delimitatori diversi (Punto e Virgola, Tab)

Spesso i CSV generati in Europa (es. da Excel italiano) usano il punto e virgola ; come separatore per non confondersi con la virgola dei decimali (3,14). Cambiare il parsing è semplicissimo:

  • In cut: cut -d';' -f1
  • In while: while IFS=';' read -r ...
  • In awk: awk -F';' '{print $1}'

Per i file TSV (Tab-Separated Values), il delimitatore di default di awk (che è lo spazio/tab) funziona già perfettamente. Per cut, il tab è il delimitatore di default, quindi basta usare cut -f2 file.tsv.

9. CONTROLLO DEI PROCESSI, SEGNALI E INTERFACCIA GRAFICA (TUI)

Gestione Asincrona (&) e Sincronizzazione (wait)

Lanciando un comando seguito dal carattere &, la shell esegue una fork() ed inserisce il processo figlio in background asincrono, restituendo immediatamente il controllo al terminale.

  • $!: Cattura immediatamente il PID del job asincrono appena generato.
  • wait [PID]: Blocca l'esecuzione dello script padre finché il processo figlio specificato non ha completato l'esecuzione.

Intercettazione dei Segnali del Sistema Operativo (trap)

Il comando trap permette allo script di registrare una funzione di callback personalizzata per rispondere alla ricezione di segnali asincroni inviati dal Kernel (es. SIGINT generato da Ctrl+C o SIGWINCH generato dal ridimensionamento della finestra).

cleanup() {
    tput cnorm # Ripristina il cursore visibile prima di morire
    clear
    exit 0
}

# Registrazione dei trap: Associa la funzione cleanup ai segnali di interruzione e terminazione
trap cleanup SIGINT SIGTERM

Gestione Avanzata del Terminale (tput)

L'utility tput permette di interagire direttamente con le capacità dello schermo (ANSI escape sequences) per creare interfacce grafiche testuali (TUI).

  • tput lines / tput cols: Restituiscono in tempo reale il numero di righe e colonne disponibili nella finestra corrente.
  • tput cup <y> <x>: Sposta istantaneamente il cursore alle coordinate cartesiane impostate (0-indexed, dall'angolo in alto a sinistra).
  • tput civis: (Cursor Invisible) Nasconde alla vista il cursore lampeggiante di sistema.
  • tput cnorm: (Cursor Normal) Ripristina la normale visibilità del cursore del terminale.

Interfacce Interattive native (whiptail)

whiptail permette di generare finestre di dialogo grafiche all'interno della CLI sfruttando la libreria newt.

Creazione Menu di Selezione Singola (--menu)

I parametri richiedono l'altezza, la larghezza, il numero di elementi visibili e coppie composte da Tag e Descrizione. Attenzione: whiptail restituisce la scelta fatta scrivendo sullo stderr (2), non sullo stdout (1). È necessario attuare uno swap dei canali per catturare il dato in una variabile.

# Lo swap '3>&1 1>&2 2>&3' scambia stdout e stderr per permettere la cattura della variabile
SCELTA=$(whiptail --menu "Seleziona Database" 12 45 3 \
    "1" "PostgreSQL" \
    "2" "MariaDB" \
    "3" "SQLite" \
    3>&1 1>&2 2>&3)

Barra di Avanzamento Dinamica (--gauge)

La barra legge continuamente un flusso di numeri interi (da 0 a 100) provenienti dallo standard input e aggiorna l'interfaccia grafica in tempo reale, terminando quando rileva l'EOF.

Esempio 4: Array Associativi e Xargs

#!/bin/bash
# Dichiara esplicitamente un array associativo (dizionario chiave-valore)
declare -A EXT_COUNTS

# Cicla su tutti gli elementi nella directory corrente
for file in *; do
    # Se non è un file regolare (-f), salta al prossimo giro (continue)
    [[ -f "$file" ]] || continue
    
    # Estrae l'estensione: taglia (cut) usando il punto (-d.) e prende dal secondo campo in poi (-f2-)
    ext=$(echo $file | cut -d. -f2-)
    # Se la variabile è vuota (-z), assegna un nome di default
    if [[ -z $ext ]]; then ext="no_extension"; fi
    
    # Incrementa il contatore per la specifica chiave (estensione)
    ((EXT_COUNTS[$ext]++))
done

# Iterazione sulle CHIAVI dell'array associativo (nota il punto esclamativo '!')
for ext in "${!EXT_COUNTS[@]}"; do
    mkdir -p "$ext"
    # printf con '\0' separa i file con un carattere nullo, supportando file con spazi nel nome
    for file in *.$ext; do
        printf '%s\0' "$file"
    done | xargs -0 -I{} mv {} $ext  # xargs prende i nomi null-terminated e li passa a mv
done

Esempio 5: Analisi Permessi ed Esportazione Funzioni

#!/bin/bash
# Funzione per analizzare i permessi di un file
audit_file() {
    file="$1" # Il primo argomento passato alla funzione
    # Usa 'stat -c "%a"' per estrarre i permessi in formato ottale (es. 644)
    perms=$(stat -c "%a" "$file")
    # Estrae l'ultimo carattere della stringa (i permessi 'others' / world)
    world_perms="${perms: -1}"

    # Se il permesso per gli altri è >= 4 (lettura), è considerato vulnerabile
    if [ "$world_perms" -ge 4 ]; then
        echo "[WARNING] $file is world-readable!"
        # Rimuove il permesso di lettura (-r) agli altri (o)
        chmod o-r "$file" 
    fi
}
# Esporta la funzione in modo che sia visibile alle sub-shell (usate da xargs)
export -f audit_file

# Array di file sensibili da controllare
sensitive_files=("/etc/passwd" "/etc/shadow")
# Stampa i nomi dei file separati da \0, poi xargs esegue una subshell per chiamare audit_file su ciascuno
printf "%s\0" "${sensitive_files[@]}" | xargs -0 -I {} bash -c 'audit_file "{}"'

Esempio 6: Gestione Log Temporale (touch -d)

#!/bin/bash
AGE_THRESHOLD="7 days ago" # Data limite
SIZE_THRESHOLD=$((10 * 1024 * 1024)) # 10 MB calcolati in byte

REF_FILE=".age_threshold_marker"
# 'touch -d' crea un file temporaneo con la data di modifica impostata a 7 giorni fa
touch -d "$AGE_THRESHOLD" "$REF_FILE" 2>/dev/null

for log_file in *.log; do
    [ -f "$log_file" ] || continue
    # -ot (older than) confronta se log_file è più vecchio del nostro file di riferimento
    if [ "$log_file" -ot "$REF_FILE" ]; then
        # 'stat -c "%s"' ottiene la dimensione del file in byte
        file_size=$(stat -c "%s" "$log_file")
        if [ "$file_size" -gt "$SIZE_THRESHOLD" ]; then
            rm "$log_file" # Elimina se maggiore di 10MB
        else
            gzip "$log_file" # Comprime se minore o uguale a 10MB
        fi
    fi
done
# Pulizia del file temporaneo di riferimento
rm -f "$REF_FILE"

Esempio 7: Controllo Processi (pgrep, renice)

#!/bin/bash
# Prende il nome del processo come parametro dallo script
PROCESS_NAME="$1" 
# Esegue pgrep -i per cercare i PID dei processi, ignorando il case
pids=$(pgrep -i "$PROCESS_NAME")

# Itera attraverso la lista di PID trovati
for pid in $pids; do
    # Verifica silenziosamente se il processo esiste ancora
    if ! ps -p "$pid" > /dev/null 2>&1; then continue; fi
    
    # Richiede all'utente di scegliere la priorità
    read -p "Select priority for PID $pid [1) High, 2) Low]: " choice
    
    # Analizza la scelta dell'utente per impostare il valore 'nice'
    case "$choice" in
        1|"[Hh]igh") new_nice="-10" ;; # Priorità alta (valore negativo)
        2|"[Ll]ow")  new_nice="10"  ;; # Priorità bassa (valore positivo)
        *) continue ;;                 # Salta se la scelta non è valida
    esac
    
    # Applica la modifica della priorità al processo specificato
    renice -n "$new_nice" -p "$pid" > /dev/null 2>&1
done

Esempio 9: Monitoraggio SysFS

#!/bin/bash
CUR_FREQ_FILE="/sys/devices/system/cpu/cpu0/cpufreq/scaling_cur_freq"
OLD_FREQ=$(cat "$CUR_FREQ_FILE")

while true; do
    CUR_FREQ=$(cat "$CUR_FREQ_FILE")
    STABLE_THRESHOLD=$(( OLD_FREQ / 10 ))
    
    if (( CUR_FREQ <= OLD_FREQ + STABLE_THRESHOLD && CUR_FREQ >= OLD_FREQ - STABLE_THRESHOLD )) ; then
        TREND="Stable"
    elif (( CUR_FREQ > OLD_FREQ )); then
        TREND="Increasing"
    else
        TREND="Decreasing"
    fi

    printf "Frequenza: %s kHz | Trend: %s\n" "$CUR_FREQ" "$TREND"
    OLD_FREQ=$CUR_FREQ
    sleep 1
done

Esempio 11: Parsing Configurazione (sed)

#!/bin/bash
process_file() {
    local file_path="$1"
    
    sed -e 's/#.*//' -e 's/^[[:space:]]*$//' -e '/^$/d' "$file_path" | while read -r line; do
        include_target=$(echo "$line" | sed -n "s/^include[[:space:]]\+\(.\+\)/\1/p")
        
        if [[ -n $include_target ]]; then
            process_file "$include_target" 
        else
            echo "$line"
        fi
    done
}
process_file "$1"
